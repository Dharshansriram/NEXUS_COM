# Quick Fix Checklist - Component Compass npm Error

## The Problem
```
npm error code ETARGET
npm error notarget No matching version found for @radix-ui/react-dropdown-menu@^2.2.15.
```

## Quick Solution (5 minutes)

### Option A: Automated (Easiest) ✓ RECOMMENDED
1. Open PowerShell **as Administrator** (Right-click → Run as Administrator)
2. Navigate to your project folder
3. Run: `.\fix-installation.ps1`
4. Wait for completion ✓

### Option B: Manual Fix (10 minutes)

1. **Open PowerShell as Administrator**

2. **Go to your project:**
   ```powershell
   cd C:\Users\Dharshansriram\Downloads\component-compass-main\component-compass-main
   ```

3. **Clean everything:**
   ```powershell
   rm -r node_modules -Force
   rm package-lock.json
   npm cache clean --force
   ```

4. **Fix package.json** - Open with Notepad and change:
   - Line 30: From `"@radix-ui/react-dropdown-menu": "^2.2.15",` 
   - To: `"@radix-ui/react-dropdown-menu": "^2.1.2",`
   
   - Line 28: From `"@radix-ui/react-context-menu": "^2.2.15",`
   - To: `"@radix-ui/react-context-menu": "^2.1.5",`

5. **Install again:**
   ```powershell
   npm install
   ```

6. **If still fails, use legacy mode:**
   ```powershell
   npm install --legacy-peer-deps --force
   ```

## What These Versions Mean

| Term | Meaning |
|------|---------|
| `^2.1.2` | Can use any version from 2.1.2 to <3.0.0 |
| `2.2.15` | This exact version (doesn't exist on npm) |
| `~2.1.2` | Can use 2.1.x versions only |
